// --- 3D COMPONENT: The Wireframe Globe ---
const WireframeGlobe = () => {
  const meshRef = useRef<any>(null);

  // Auto-rotate the globe
  useFrame((state, delta) => {
    if (meshRef.current) {
      meshRef.current.rotation.y += delta * 0.2; // Slow rotation
      meshRef.current.rotation.x += delta * 0.05; // Slight tilt
    }
  });

  return (
    <mesh ref={meshRef} scale={2.5}>
      {/* args: [radius, widthSegments, heightSegments] 
         Lower segments = more "retro/digital" look 
      */}
      <sphereGeometry args={[1, 16, 16]} /> 
      <meshBasicMaterial 
        color="#22d3ee" // Cyan (Tailwind's cyan-400) - MATCHES YOUR ACCENT
        wireframe={true} 
        transparent 
        opacity={0.3}

  );
};

### **Task for your Coding Partner (Theme Behavior)**

Since you switch between "Light Mode" and "Terminal Mode", this brutalist style (which relies on borders) needs careful handling.

**Please forward this task list to your partner:**

1. **Light Mode Handling:**
* In `Light Mode`, the `border-accent/20` might look too faint or too harsh depending on the background.
* **Task:** Add a condition or a specific Tailwind class for light mode (e.g., `light-mode:border-gray-300`) to ensure the grid lines remain visible but elegant.


2. **3D Globe Color:**
* Currently, the Globe is hardcoded to Cyan (`#22d3ee`).
* **Task:** Pass the current theme color to the `<WireframeGlobe />` component so the globe turns Dark Blue or Black when in Light Mode (for better contrast).


3. **Font Weight:**
* The "Acid" style uses `font-black` (very thick).
* **Task:** Ensure the font loads correctly. If `Inter Black` or `Oswald` isn't available, the fallback might look weak. Consider adding `className="font-black"` to the global CSS or importing a heavier weight from Google Fonts.
